1. Health & Sensor Data — Detailed Disclosure

Google Play requires us to clearly disclose any access to health-related or sensor data. The following section provides a full and transparent account of all such data the App accesses.

1.1 Motion & Orientation Sensor Data

The App accesses the device's accelerometer, gyroscope, and magnetometer (compass) sensors exclusively to calculate and display the Qibla (prayer direction).

  • Data accessed: Raw sensor readings — device orientation, heading angle, tilt.
  • Purpose: Qibla compass calculation only.
  • Processing location: 100% on-device, in real time.
  • Stored? No. Sensor readings are never written to disk, a database, or any server.
  • Shared? No. Sensor data is never transmitted to us or any third party.
  • Retention period: Zero — data exists only in RAM while the Qibla screen is open.

✅ No health or fitness data is collected. Sensor access is limited to orientation/compass readings for religious direction calculation. The App does not access step counts, heart rate, blood oxygen, sleep data, menstrual cycles, nutrition, weight, glucose, or any other health metric available on the device.

1.2 Why Sensor-Related Packages Are Present (Technical Clarification)

The App uses expo-sensors from the Expo SDK framework. This package is a general-purpose sensor library that can access multiple sensor types; however, the App only invokes compass/magnetometer APIs within this library. No other sensor API provided by expo-sensors is called or used.

⚠️ The presence of expo-sensors in the app package does not mean all sensor types are accessed. Only the magnetometer/orientation APIs are invoked. This is verifiable in our source code.

1.3 Physical Activity / Step Counter

The App does not request the ACTIVITY_RECOGNITION permission (Android) or the NSMotionUsageDescription entitlement (iOS) and therefore cannot access step count or physical activity data.

2. All Data Accessed or Collected

2.1 Location Data

  • Type: Approximate and/or precise GPS coordinates.
  • Purpose: Determine city for prayer time API requests; calculate Qibla direction.
  • Storage: City name only is stored locally on-device (AsyncStorage). Raw GPS coordinates are never persisted.
  • Shared with third parties: City/region name is sent to a prayer time API solely to retrieve prayer schedules. No raw GPS is transmitted.
  • Retention: Stored city name is kept until the user changes it or uninstalls the App.

2.2 Notification Data

  • Type: Locally scheduled notification triggers (prayer times, religious days, user reminders).
  • Storage: Stored locally on the device via expo-notifications. Not transmitted to any server.
  • Retention: Until the user cancels the reminder or uninstalls the App.

2.3 User Preferences (Local Storage)

  • Selected city, reminder settings, UI preferences.
  • Stored locally via AsyncStorage. Never transmitted externally.
  • Retention: Until modified by the user or App is uninstalled.

2.4 Advertising Data (Google AdMob)

The App displays advertisements via Google AdMob (react-native-google-mobile-ads). AdMob may independently collect:

  • Advertising ID (GAID / IDFA)
  • Device identifiers and model information
  • Ad interaction data (impressions, clicks)
  • IP address (for ad targeting)

This data is collected and processed by Google under Google's Privacy Policy. We do not receive or store this data ourselves.

4.2 What Happens When You Uninstall the App

When you uninstall the Prayer Time App from your device, all locally stored data is permanently and automatically deleted by the operating system. This includes:

  • All AsyncStorage entries (city, preferences, reminder settings)
  • All locally scheduled notifications
  • All cached Quran content
  • Any other app-specific files stored in the app's sandboxed storage

✅ Because all data is stored exclusively on your device (with the exception of AdMob data which is governed by Google), uninstalling the App fully removes all data associated with your usage. No server-side deletion request is necessary.

4.3 How to Delete Specific Data Without Uninstalling

  • Prayer time reminders: Open the App → Settings → Notifications → disable individual or all prayer notifications.
  • Religious day reminders: Open the App → Special Days → tap the reminder you wish to remove → Delete Reminder.
  • City / location preference: Open the App → Settings → City → select a new city or clear the selection.
  • Location permission: Device Settings → Apps → Prayer Time → Permissions → Location → Deny.
  • All local app data (Android): Device Settings → Apps → Prayer Time → Storage → Clear Data.
  • All local app data (iOS): Device Settings → General → iPhone Storage → Prayer Time → Offload App (or Delete App).
  • AdMob advertising ID: Device Settings → Google → Ads → Reset advertising ID / Opt out of Ads Personalization.

4.4 Data Deletion Requests

Since we do not collect or store any personal data on external servers, there is no server-side deletion action for us to perform. All data resides exclusively on your device and is deleted automatically upon uninstallation.

If you have any questions or concerns about data deletion, please contact us at the email address listed in Section 10 of this policy and we will respond within 30 days.
